Beyond the basics with Claude Code

Daisy Holman · Anthropic(Claude Code,前 C++ 委员会主席)

开箱即用的 Claude Code 处理「编程任务」,但真正大规模的软件工程需要你定制 agentic harness,让 Claude 够到你够到的一切。硬约束是固定的上下文窗口——选能扩展到 10 万+ skill 的抽象。

  • agent 干你的活要三样:access、knowledge、tooling;测试:不离开 Claude 终端干一整天,每次 alt-tab 都是一处缺失的连接。
  • 别为代码规范/制度记忆 fine-tune(对churning 的前沿模型不划算,还可能增加幻觉);用 in-context learning =「文本文件的高级说法」。苦涩的教训:通用 AI 打败专用 AI。
  • 用 post-tool-use hooks 给 agent 做「红波浪线」(linter/LSP):「让 agent 变好最快的不是更聪明的模型,是更紧的反馈回路」。
  • KV 缓存:改早期 token 会让整段下游缓存失效(未缓存 ~10× 成本)→ 稳定内容放前面,易变的 per-task 放后面。
  • 四种插件原语按可扩展性:MCP(撑大系统提示,只给非技术用户用)、skills(惰性 body、description 常驻)、hooks(零上下文开销,最佳)、subagents(独立上下文,但 description 仍占父提示)。
  • 异步/并行工作流:git worktree + 每个一个常驻 Claude、Claude 间消息、/loop 看 PR/CI、auto mode(~+30-40% token)跑通宵、agents 面板 + 远程遥控。
数字Opus 4.7 ~1M token 上下文(近一年保持 ~1M);未缓存 ~10×;auto mode +30-40% token;一个工程师靠 agents 视图一月 ~1000 PR;有公司在 monorepo 里撞到 1万-10万 skill 的天花板。
金句“如果 Claude 不能做你能做的一切,它就没法和你一起做你的工作。”
So-what把 Claude 接进你用的每一个信息源,并用「如果有 10 万个会怎样?」来选定制原语(优先 hooks,慎用 MCP/CLAUDE.md)——上下文窗口才是稀缺资源。

Build a proactive agent workflow(Routines)

Maya · Anthropic Applied AI

「编程 agent 不该等你按回车。」新的 routines 把 Claude Code 从被动工具变成主动队友:跑在托管基础设施上,按计划或事件触发,同时仍然可交互、可引导。

  • DIY 的痛:在哪跑(笔记本一关就死)、托管/持久化/鉴权样板、触发方式、headless 会话看不见也插不上手。
  • Routines:只定义 prompt + 仓库 + 连接器 + 触发器,Claude Code 负责托管、会话状态、连接器鉴权。
  • 三目标:始终在线;主动(时间表/原生 GitHub 事件/自定义 webhook,payload 当上下文);可交互可引导(每个 routine 都是真会话,可从 web/CLI/桌面打开、观看、引导、续跑)。
  • 三决策:触发器(何时)、上下文(成功的天花板)、可引导性(怎么保持它不跑歪)。
  • 内部案例:Claude Code 周 PR 自年初 +200% 压垮了单个文档维护者 → 她用 /schedule 每周 diff 主干 vs 文档、自动开更新 PR。
  • 可引导模式:agent 互审(生成器/批评者)、实时人类介入、输出验证(渲染改后的页面确认)。
数字Claude Code 周 PR 自年初 +200%;触发器:计划/GitHub 事件/webhook;连接器:GitHub、Slack、Google Drive、Datadog、Grafana、Twilio。
金句“我们想让 Claude 从一个工具,变成一个队友。”
So-what用 Claude Code routines 替掉你为后台 agent 手搓的 cron + 托管:定义 prompt + 仓库 + 连接器 + 触发器,再加 agent 互审或实时引导保质量。

Stop babysitting your agents

Sidh Bhattacharya · Claude Code 创始工程师 · Anthropic

模型越聪明,工程师越在浪费时间当「人肉 QA」。夺回这些时间靠叠三样能力:验证循环、多开 Claude、后台循环——让 Claude 可靠地自查自己的活、不用你守在键盘前。

  • 前置(「Claude Code 301」):高质量 CLAUDE.md(单点杠杆最高)、连上你的工具/MCP(Slack/Linear/Datadog/BigQuery)、配好远程环境(算力和笔记本解耦)。
  • 为 agent(不是人)重构工具:「你的代码库里,有什么是人类视为理所当然、而 agent 需要的?」
  • 验证 = 教 Claude 走人类那套:写码 → build/typecheck → 跑起来 → 查副作用(浏览器/日志/DB)→ 单测 → staging。循环 = 写、查失败、debug、重复到成功态。
  • 前端循环四件套:跑 app(dev server)+ 驱动浏览器(Claude-in-Chrome /chrome 或 Playwright)+ 用前后截图证明 + 处理鉴权/状态。把循环打包成会自我记录的 skill(每次卡住就写文档)。
  • 多开 Claude(注意力是稀缺,他上限 ~4-5):桌面 app 侧栏、终端 Claude Agents(按需注意排序)、云(claude.ai/code)、手机远程遥控(/remote-control,他最爱)。
  • 后台循环:/loop 按间隔跑(「/loop 10 分钟,看着我的开 PR」);Routines = /loop 跑在云容器里,时间或事件触发。
数字现场:几乎人人每天用 Claude Code,~50% 配过 CLAUDE.md + 连了工具;Claude Agents ~一周前发布;验证循环搭起来 ~5-10 分钟;个人同时 ~4-5 个会话。
金句“把你的键盘,移出热路径。”
So-what给 Claude 工具和指令去闭合一个验证循环,打包成会自我改进的 skill,再把例行活(PR/文档/triage)扔给 /loop 和 Routines,让你稀缺的注意力只去该去的地方。

What's new in Claude Code

Ralf · Anthropic 技术团队(对接欧洲伙伴/客户)

最近发布的两条主线:开发体验(让工具用着舒服)和自治(让 Claude 在不需要时别打断你、自己多干)。

  • 远程遥控:电脑上起会话,手机 app/任意浏览器接着用,需要输入时收通知(settings.json 设成常开)。
  • 无闪烁全屏 TUI(/tui full-screen):虚拟化滚动区(内存恒定),支持可点/可展开元素。
  • 桌面端刷新:会话按项目分组,看 plan + 行内评论(推回 Claude)、PR diff、逐行评论、GitHub 集成。
  • auto mode:分类器逐动作判断「破坏性/会后悔吗 + 像 prompt 注入吗」,安全就自己干,不安全找绕法,只在没有安全路径时才问你。
  • 原生 worktree(claude-worktree):每会话把仓库复制到隔离子目录,并行 Claude 不打架。
  • auto memory:Claude 干活时自动记笔记进 memory.md(干活的笔记)≠ CLAUDE.md(入职手册);短索引 + 渐进披露;跨会话共享、留本地、/memory 可审。
  • code review:每个 PR 上多 agent 两阶段(找 bug/漏洞 → 对真实代码验真降误报),GitHub app 原生或 /ultra-review。
  • Routines(研究预览)+ Agent View(公测,claude agents:所有会话按状态分组,空格 peek/发指令不进入)。
数字auto mode = 两查分类器;命令:/ultra-review、/tui full-screen、claude agents。企业:更好的 Windows 支持、GCP/AWS 部署向导、治理用原生二进制。
金句“现在 Claude Code 就是终端。”
So-what打开那些自治特性(auto mode、auto memory、原生 worktree、远程遥控),把评审和例行活交给 /ultra-review 和 Routines,这样你能异步跑 Claude、只在它真需要时介入。
