Building with Claude on Google Cloud

Iman Nardini · Google Cloud(与 Anthropic 合作)

Claude Code(跑在 Google Cloud 托管的 Claude 模型上)能带一个非专家走完从想法到部署的整个生命周期——UI、后端、安全评审、分析——在 Google Cloud 上建并部署一个真 app,而无需懂 Google Cloud。

  • 上手简单:用应用默认凭据(ADC,无需存/轮换 API key);向导检测项目/区域、核对可用的 Claude 模型。
  • 五角色 demo,一职能一顶帽子:PM(草图→线框)、UI/UX(线框→4 个页面)、SWE(后端+部署)、安全(评审→上生产)、分析(指标→看板)。
  • 展示 Claude Code 特性:CLAUDE.md 角色文件、plan mode、subagents(并行模拟团队冲刺)、MCP、skills、自定义插件。
  • 两个新 GCP 集成:官方 Google Cloud Skills + Developer Knowledge API/MCP(文档每 24h 刷新)。
  • 落地架构:Cloud Run(API,serverless)+ Firestore(库)+ BigQuery(分析仓)+ Looker(看板);CI/CD 由 Claude 用 Cloud Build + Cloud Deploy 自动建(PR 合并触发,先 dev 再手动 promote 到生产)。
  • 安全评审用自定义插件(公司规则,OWASP 式 + 输入校验 + 最小权限);Claude 修复、重跑测试、开新 PR。
数字按 token 计费(无消息上限);可选 provisioned throughput;文档每 24h 刷新;多区域端点。产品:Cloud Run、Firestore、BigQuery、Looker、Cloud Build、Cloud Deploy。
金句“你居然在不懂 Google Cloud 的情况下,把东西部署到了 Google Cloud 上。”
So-what用 ADC 认证 + Developer Knowledge MCP + 官方 GCP skills,Claude Code 能设计并部署一整套 Cloud Run/Firestore/BigQuery/Looker + CI/CD,哪怕你没有 GCP 经验。

Build AI agents using Claude in Microsoft Foundry

Marlene Mangami 等 · Microsoft(动手 workshop)

动手实操:在 Microsoft Foundry 部署 Claude → 用 Microsoft Agent Framework 把模型接成 agent → 用 MCP server 给它真工具——最后做出一个会下单蛋糕的「Sparkles」agent。

  • 立意:行业从单轮 chat 转向会规划/推理/行动的 agentic 系统;三挑战:长上下文多步推理、企业级可靠/可观测/安全、连外部工具与数据。「不能光等模型变好,得有系统把智能执行出来」。
  • Foundry = 统一平台(配 GitHub、VS Code、Visual Studio、Copilot Studio);产品:Foundry 模型(含 Claude)、Agent Service、工具/集成、ML 服务。
  • 选 Claude + Foundry 的四个理由:最佳 agent 推理(Opus 当主力)、能编排 agent 并给工具、企业能力开箱(Defender/Purview/Entra ID,「不用老跟 .env 打交道」)、原型到生产更快(内置 eval/可观测/监控)。
  • 流程:Foundry 部署 Claude → Playground 测 → 进 VS Code(.env 的 URI 要以 anthropic 结尾、去掉末尾 v1/messages)→ 用 Microsoft Agent Framework(Python)建 agent → 接蛋糕店 MCP server 当工具 → 从 MCP 拉人设/欢迎语等可复用 prompt。
  • 要点:MCP 暴露的不只是工具,还有 prompts 和 resources。
数字Foundry 自带 1,400+ 连接器/MCP 工具;模型:Claude Sonnet 4.6(workshop 用)、Opus 4.7;集成:Defender、Purview、Entra ID、SAP、ServiceNow。
金句“用 MCP,你只要一个 URL,就拿到对 agent 友好的整套 API。”
So-what要把 Claude agent 装进微软/Azure 企业栈:Foundry 部署模型、Microsoft Agent Framework 搭 agent,用 MCP server 提供能力(工具 + 可复用 prompt + resources),而不是硬编码。

AI with Claude on AWS: From code to orchestration

Antonio Rodriguez · AWS(自称「Claude whisperer」,动手 workshop)

「better together」+ 实操:把 Claude app 从本地原型搬上 AWS 生产——讲清 AWS 上用 Claude 的三种方式,以及安全/计费/可扩展的好处。

  • AWS–Anthropic 合作:Amazon 数十亿投资、AWS 为主云、Anthropic 承诺 1000 亿+ AWS 用量;Project Rainier 是最大 AI 算力之一;Trainium 第三代。
  • 三个技术角度:Bedrock 全平台(AWS 是唯一能在云上 fine-tune Haiku 的)、安全(数据留你的 AWS 边界、最新推理引擎零运营方访问、不拿你数据训练)、近乎无限扩展 + 区域选择(可只在伦敦/欧盟跑以合 GDPR)。
  • Bedrock 生态:评估、prompt 优化、fine-tune、蒸馏、托管知识库(RAG)、Guardrails(内容过滤/禁topic/PII 脱敏/幻觉检查)。
  • Bedrock AgentCore:安全的 agent 托管,兼容 LangChain、CrewAI、Claude Agent SDK。
  • 三种用法:① Bedrock 直接调 Claude(刚加 Opus 4.7 + Messages API)② Claude Platform on AWS(几天前 GA,直连 Anthropic 体验、AWS 当网关、功能对齐)③ Claude Desktop/Cowork/Code 走 AWS 合并账单(Marketplace 或新 Claude 3P 连接器)。
  • workshop:用 Excalidraw 仓库让 Claude Code 画/改架构图;模块从基础 → 上下文工程 + Playwright MCP 截图 + git → 子 agent/插件/skill/hook。
数字数十亿投资 / 1000 亿+ 用量承诺;Trainium 三代;Bedrock 上 Opus 4.7 + Messages API;Claude Platform on AWS 已 GA;零运营方访问。合规:FedRAMP、HIPAA。
金句“如今我们基本是在「轻语」模型,而不是真在写代码。”
So-what已经在 AWS 上的话,可三种方式跑 Claude(Bedrock API、刚 GA 的 Claude Platform 网关、或 Desktop/Code 合并计费),拿到企业安全/私网/可观测,并在 AgentCore 上托管 Agent SDK——不出你现有的合规/计费边界。
