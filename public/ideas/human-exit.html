人类退场。

把这几篇 /ideas 分开看,是几条独立的新闻;串起来看,是同一个故事 —— AI 时代,人正从一个个位置上退场。 问题已经不是「会不会」,而是「退到哪、剩下什么」。这一辑按退场的层层深入排: 从公司缩到几个人,到人只当 exception owner,到工程师不再写代码,直到把人移出信任链、只剩价值观。

4 篇 组织 → 运营 → 手艺 → 综合 策划视角:同一条退场线
§ 一条退场线 · 越往右,人退得越深
组织层
公司缩到几个人
团队从一整支,缩到刚好还能保证体验的几个人。
运营层
人只当 exception owner
Agent 跑 happy path,人退到只守搬实物、关系、对钱负责。
手艺层
工程师不写代码
人退出「写」的位置,改成写 Loop、编排 agent。
综合层
移出信任链,剩价值观
机器自己写测改、还更安全,人只剩「想让它成为什么」。
§ 一辑四篇 · 按退场深度排
组织层 · #07
退场 ①
退的是:整支团队 → 几个人

不是一人公司,是几个人的公司

Gumroad 创始人 Sahil Lavingia:真要把体验做好,最后大概不是 1 个人,而是几个人。极简不是把人减到 0,是减到刚好还能保证体验的那几个。退场的第一层,发生在组织规模上。

运营层 · #05
退场 ②
退的是:日常操作 → 只守异常

用 AI Agent 运营一家公司 · Distributor 架构

一个 Orchestrator 调度 4 个 Agent(客服 / 供应链 / 财务 / WMS),人只守三件事:搬实物、维护关系、对钱和异常负责。原则是「Agent 跑 happy path,人当 exception owner」。退场推进到运营层。

→
手艺层 · #08
退场 ③
退的是:写代码 → 写 Loop

连「品味」也不再是人类独有 · Boris Cherny

Claude Code 创造者:~200 个后台 Claude、11 月卸载 IDE、工程师从「写代码」退到「写 Loop」。他更狠的判断是 —— 连「产品品味」这道护城河,3–6 个月也会破。退场打到手艺的核心。

综合层 · #09
压轴
退的是:整条信任链上的人

把人,安全地移出信任链 · Claude Code 一周年 × Boris

团队复盘 + Boris 合看的专题。三幕:退场(人怎么一步步被移走)→ 为什么敢退(验证 · classifier · 把错误写进记忆)→ 退场之后(连品味也退场,剩下价值观)。这一辑的收口。

→

退场不是结局,
是换了个位置。

四篇连起来读,退场是一条单向加深的线:组织 → 运营 → 手艺 → 信任链。每一层,人都从「亲自做」退到「定方向、守异常、对结果负责」。 而退到尽头,这一辑给的答案不是「人没用了」,而是 —— 留给人的,从「会做」变成了「想让它成为什么」。 这也是我把它们收成一辑的原因:单看是切片,合看才是趋势。
